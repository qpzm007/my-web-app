# JKP MES GO v2 통합 기능 및 운영 명세서 (features.md)

JKP MES GO 소프트웨어는 단순한 모니터링을 넘어, 다중 프로젝트 관리, BOM 기반 공정 설계, AI 납기 시뮬레이션, 그리고 철저한 권한 기반(RBAC) 작업자 통제 시스템까지 갖춘 차세대 스마트 공장 종합 관리 플랫폼입니다.

---

## 🌟 핵심 신규 기능 (최신 업데이트 내역)

### 1. 권한 기반 접근 (RBAC) 및 전용 UI 분리
- **로그인 게이트웨이**: 모든 사용자는 접근 권한에 따라 화면이 분류됩니다.
- **권한별 기능**:
  - **마스터 & 관리자 (Master/Manager)**: 중앙 대시보드, 프로젝트 관리, 인력/계정 생성, 작업 지시 하달 권한을 가집니다.
  - **작업자 (Worker)**: 데이터 대시보드 대신, 철저히 모바일 현장에 맞게 최적화된 **개인 맞춤형 작업 통제 화면**만 제공받습니다.

### 2. 현장 작업자 전용 통제 앱 (Mobile-Friendly Worker UI)
- **작업 내용 사전 확인**: 관리자가 할당한 공정에 대해 '대기 중' 상태에서도 공정 설명 및 단위 소요량을 즉시 파악할 수 있습니다.
- **직관적 생애주기 버튼**: 
  - `[작업 시작]`: 터치 시 상태 활성화.
  - `[일시 정지]`: 식사, 휴식 등으로 인한 휴게시간 체킹.
  - `[관리자 호출]`: 즉시 관리자 대시보드로 알림 전송.
  - `[불량 보고]`: 불량 발생 시 카메라 촬영창 호출 및 내용 전송.
- **원터치 실적 등록 시스템**: 
  - `[할당량 작업 완료]` 클릭 시 양품 수량(기본값: 목표수량 100%)과 불량 수량(기본값: 0)이 자동 기입된 팝업이 출력됩니다.
  - 별도 타이핑 없이 `확정` 버튼 한 번으로 실적을 등록하고 다시 대기 상태로 전환할 수 있습니다.

### 3. 다중 프로젝트(Project) 및 공정(BOM) 관리
- **스위치 지원**: 우측 상단 드롭다운을 통해 프로젝트(예: 라인 A, 라인 B)를 넘나들며 독립적인 현황 관리가 가능합니다.
- **강력한 데이터 무결성 (Safety Lock)**: 이미 작업자가 등록되어 있거나 작업이 진행 중인 공정과 프로젝트는 삭제 및 임의 수정이 절대 불가하도록 잠금 처리됩니다.
- **BOM 기능**: 각 프로세스는 단순 공정을 넘어 '단위 소요량(Required Amount)' 속성을 가져 정밀한 수량 파트로 동작합니다.

### 4. 시각적 공정 마인드맵 (Topology Map)
- **전체 흐름 파악**: 수많은 프로젝트와 공정, 그 하위에 연결된 작업자들을 시각적인 트리 형태로 노출합니다.
- **AI 병목/전문가 감지**: 
  - 병목 공정 노드는 빨간색 펄스 이펙트로 강조됩니다.
  - 등급(LV.9 이상)이 높은 직원은 노드에 "EXPERT(전문가)" 뱃지가 자동 부착되어 한눈에 식별됩니다.

### 5. 인공지능 납기 시뮬레이션 (AI Delivery Forecasting)
- 납기 수요 수량과 마감 일자를 입력하면, 가장 공정 시간이 긴 '병목 공정'의 사이클 타임을 기반으로 **예상 납기 완료일**을 자동 산출합니다.
- 목표일 내에 납품이 불가능할 경우, 시스템이 **필요한 추가 숙련공의 수**를 역산하여 인력 재배치를 제안합니다.

## 🛠️ 기존 핵심 기능
- **실시간 대시보드**: 정상(Normal), 지연(Delay), 정지(Stop) 상태 모니터링.
- **음성 보고 & AI 비전**: 현장에서의 보이스 및 사진을 인식하여 AI가 자동으로 재고/로그를 연동 (Gemini 연동 가능 구조).
- **공정 분석 (Analytics)**: 재고 상태 바 차트 및 평균 작업 시간 비교 차트.
- **전체 활동 로그**: 중앙 타임라인 통합 기록망.

## 💡 종합 운영 프로세스 가이드

### 단계 1: 프로젝트 및 공정 세팅 (Master/Manager)
1. 시스템 로그인 후 상단 헤더에서 활성화할 **프로젝트**를 선택하거나 신규 등록합니다.
2. [공정 및 BOM 설정]에서 각 스테이지별 이름, 단위 소요량을 설정합니다. (진행 내역이 생기기 전에 신중히 입력)

### 단계 2: 계정 발급 및 작업자 할당 (Master/Manager)
1. 하단 **[인력/계정 관리]** 메뉴에서 직원들의 로그인 ID, PW 및 직급(Manager/Worker), 숙련도를 등록합니다.
2. 마스터/관리자는 대기 중인 Worker들에게 즉석에서 "작업 지시하기" 버튼을 눌러 목표량과 공정을 할당합니다.

### 단계 3: 현장 작업자의 실행 (Worker)
1. 현장 작업자는 태블릿으로 Worker 계정으로 로그인합니다.
2. 할당된 화면을 확인하고 **[작업 시작]**을 탭합니다.
3. 작업 종료 시 **[작업 완료]** 버튼을 통해 양품/불량 수량을 즉각 모달로 보고합니다.

### 단계 4: 실시간 관제 및 AI 시뮬레이션 (Master/Manager)
1. 중앙 메뉴에서 모든 Worker의 대기/작업/정지 현황을 실시간 관제합니다.
2. [마인드맵] 탭을 열어 빨간색으로 경고등이 켜진 병목 구간을 시각적으로 파악합니다.
3. [납기 시뮬레이터]를 활용해 현 상태로 납기일을 맞출 수 있을지 가늠하고, 필요하다면 마인드맵에서 'EXPERT' 뱃지를 가진 타 부서 직원을 병목 공정에 긴급 재배치합니다.

## 🔍 누락된 기능 검토 (Review)

* 프로젝트별 진행내역 의존성 안전장치(수정/삭제 불가 Lock) ✓
* 마인드맵 공정/인원/숙련자/병목 시각화 ✓
* 공정 사이클 자동 계산 및 숙련공 투입 시뮬레이터 ✓
* 권한 구분 로그인 및 모바일 최적화된 작업자 전용 실행 UI ✓
* 작업 시작/일시정지/관리자 호출/불량 내역 보고 기능 ✓
* 대기 상태 중 상세 내역/BOM수량조회 및 작업 완료(양품/불량 실적) 모달 적용 ✓

초기 요구사항이었던 기본적인 재고/상태 관리를 전부 포괄하면서도, 더욱 거대하고 지능화된 공장 시스템 구현이 완료되었습니다.
